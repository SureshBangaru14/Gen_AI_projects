What is Similarity :

	- Similarity means measuring how much two things are alike.

	- In AI and data science, similarity is used to compare:

		Words
		Sentences
		Images
		Documents
		Products

	- Instead of exact matching, similarity helps us understand meaning and closeness.
	
	
	
Simple Definition : 

	- Similarity = a score that tells how close two items are in meaning or structure.

	Example:

		- “car” and “automobile” → very similar
		- “car” and “banana” → not similar
	
	
Types Of Similarity :


	1. 📐 Cosine Similarity (Most popular in AI) :
	
		- Measures angle between two vectors
		- Ignores size, focuses on direction
		- Used in embeddings and NLP

		✔ Best for:

			- Text similarity
			- Chatbots
			- Search engines

		Example:

			- “AI is smart” ↔ “Artificial intelligence is intelligent” → high similarity
			
			
			
		⭐ Types of Cosine Similarity

			- Cosine similarity values range from -1 to 1

			🟢 1. Positive Similarity (0 to 1) : 
			
				Means vectors are similar
				Higher value = more similarity

				Examples:

					“car” ↔ “automobile” → 0.8 to 1.0
					“dog” ↔ “puppy” → high similarity

					👉 Interpretation:
					✔ Same or related meaning
					✔ Used in search and recommendation

			⚪ 2. Zero Similarity (0) : 
			
				Means vectors are unrelated
				They are perpendicular (90° angle)

				Examples:

					“pizza” ↔ “computer”
					“cat” ↔ “finance”

					👉 Interpretation:
					❌ No relationship in meaning


			🔴 3. Negative Similarity (-1 to 0) :
			
				Means vectors are opposite in meaning
				Rare in real NLP embeddings, but possible in theory

				Examples:

					“hot” ↔ “cold”
					“increase” ↔ “decrease”

					👉 Interpretation:
					⚠ Opposite concepts


	2. 📏 Euclidean Similarity (Distance-based) :
	
		- Measures straight-line distance between points
		- Smaller distance = more similar

		✔ Best for:

			- Image recognition
			- Clustering data

		Example:

			- Two similar images have smaller distance
			
			
			
			
			
	3. 📊 Jaccard Similarity :
	
		Compares overlap between two sets
		Formula: intersection / union

		✔ Best for:

			Recommendation systems
			Keyword matching

			Example:

			Set A = {apple, banana}
			Set B = {banana, mango}
			→ similarity based on common items
			
		
	
	4. 🔤 Edit (Levenshtein) Similarity :
	
		Measures how many changes needed to convert one word into another

		✔ Best for:

			Spell check
			Typo correction

			Example:

			“kitten” → “sitting” (few edits needed)


	5. 🧠 Semantic Similarity (AI-based) :
	
		Understands meaning, not just words
		Uses embeddings

		✔ Best for:

		ChatGPT-like systems
		Semantic search

		Example:

		“I feel happy” ↔ “I am joyful” → similar meaning
		


	Simple Way to Remember :
	
		Cosine → meaning-based text search
		Euclidean → physical distance (images/face)
		Jaccard → set overlap (likes/items)
		Edit → spelling correction
		Semantic → deep understanding like humans

What is Embedding : 

	- Embedding is a way to convert words, sentences, images, or data into numbers (vectors) so that a computer can understand their meaning.

	- Instead of reading text like humans do, machines understand numbers, so embeddings translate meaning into a mathematical form.

	- An embedding is a numerical representation of data (like text) in a high-dimensional vector space, where similar meanings are placed closer together.

	For example:

		“king” → a list of numbers like [0.12, -0.44, 0.98, ...]
		“queen” → a very similar list of numbers
		“apple” → a very different list of numbers

		So the model understands:

			👉 king and queen are related
			👉 apple is unrelated

Word2Vec :

	- Word2Vec is a technique used to convert words into numerical vectors so that machines can understand word meaning based on context.
	
	- Previous techniques (like One-Hot Encoding, Bag of Words, TF-IDF) do not capture semantic relationships or contextual meaning between words; they only capture frequency-based or occurrence-based information, whereas Word2Vec learns word meanings from context and represents words as dense vectors where semantically similar words are placed closer together.
	
	- 
	
	What Word2Vec Does and It learns : 

		Words that appear in similar contexts → get similar vectors

		So:

		“king” ≈ “queen”
		“car” ≈ “vehicle”
		“doctor” ≈ “nurse”
		
		Ex :
		
		
			1. Previous Techniques (BoW / TF-IDF)
			
				Sentence 1:

				“I love car”

				Sentence 2:

				“I love vehicle”

				Bag of Words representation:
				
				car     → 1
				vehicle → 1
				love    → 1
				I       → 1
				
				Problem:
				
					“car” and “vehicle” are treated as completely different words
					Model does NOT understand they mean similar things

					👉 So similarity = ❌ NOT detected


			2. Word2Vec

				Now Word2Vec looks at context:

				Training sentences:
				
					I drive a car
					I drive a vehicle
					
				Learned vectors (example idea) :
				
				car     → [0.21, 0.88, ...]
				vehicle → [0.20, 0.85, ...]
				
				Result:
				
					“car” and “vehicle” appear in similar contexts (“drive”)
					So their vectors become close to each other

					👉 Similarity = ✅ DETECTED
					
					
			⚡ One-line conclusion:

				👉 Traditional methods treat words independently, while Word2Vec understands meaning by learning from surrounding words, making similar words like “car” and “vehicle” 	 closer in vector space.
				
				
	Word2Vec has two main types based on how it learns word relationships :
	
				1. CBOW (Continuous Bag of Words)
				2. Skip-gram
				
				
				
				🔹 1. CBOW (Continuous Bag of Words)
				
					👉 Idea:

						- Predict the center word using surrounding context words

					📌 Example:
					
						Context: "I love ___ in the morning"
						Target:  "coffee"
					
					🧠 How it works:
					
						Input = surrounding words
						Output = missing word
						
					✅ Best for:
					
						- Small datasets
						- Faster training
						
						
				🔹 2. Skip-gram
				
					👉 Idea:

						- Predict the context words using the center word

					📌 Example:
					
						Input: "coffee"
						Output: "I", "love", "in", "the", "morning"
						
					🧠 How it works:
					
						Input = one word
						Output = surrounding words
						
					✅ Best for:
					
						Large datasets
						Better for rare words
						
						
				⚡ One-line summary : 

					👉 Word2Vec has two types: CBOW (predict word from context) and Skip-gram (predict context from word), both used to learn word embeddings from text.
															
															
	🔷 Word2Vec Code Flow :											
	
			1. Collect Text Data
			   (corpus / sentences)

				↓

			2. Text Preprocessing
			   - Lowercase
			   - Remove punctuation
			   - Tokenization

				↓

			3. Build Vocabulary
			   - Assign index to each word

				↓

			4. Generate Training Data
			   CBOW → (context → target)
			   Skip-gram → (target → context)

				↓

			5. Train Word2Vec Model
			   (Neural Network learns embeddings)

				↓

			6. Get Word Vectors
			   Example:
			   "king" → [0.25, 0.88, ...]

				↓

			7. Similarity Calculation
			   - Cosine similarity
			   - Find similar words

				↓

			8. Downstream Applications
			   - Search
			   - Recommendation
			   - Clustering

all-MiniLM-L6-v2 : 

	
	- all-MiniLM-L6-v2 is a sentence embedding model, not a text generation model like GPT. 
	   Its job is to convert text (sentences or short paragraphs) into numerical vectors (embeddings) that capture the semantic meaning of the text.

        - Breaking down the name:
        
		all – Trained on many different datasets, making it useful for general-purpose text understanding.
		MiniLM – A lightweight Transformer architecture that is much smaller and faster than models like BERT while retaining good performance.
		L6 – The model has 6 Transformer layers ("L6" = Layer 6).
		v2 – The second improved version of the model.


	Typical use cases :

		This model is widely used for :

			🔍 Semantic search
			📄 Document retrieval
			🤖 Retrieval-Augmented Generation (RAG)
			📚 FAQ matching
			💬 Chatbot knowledge retrieval
			🏷️ Text clustering
			🔖 Duplicate detection
			📝 Recommendation systems	
	
	
	For example, in a RAG system :
	
		
		
		                Documents
				    │
				      ▼
			all-MiniLM-L6-v2
				    │
				      ▼
			  Document Embeddings
				    │
				      ▼
			     FAISS Index
				      ▲
				    │
		User Question ──────┘
		      │
		         ▼
		all-MiniLM-L6-v2
		      │
		         ▼
		Query Embedding
		      │
		         ▼
		FAISS Similarity Search
		      │
		         ▼
		Top-k Documents
		      │
		         ▼
		Create Prompt

		Context:
			- Doc 1
			- Doc 2
			- Doc 3

		Question:
			...
			      │
			         ▼
			     GPT
			      │
			         ▼
			Final Answer	
			
			
	Example Code as per flow : 
	
	
		# ==========================================
		# Simple RAG using:
		# all-MiniLM-L6-v2 + FAISS + GPT
		# ==========================================


		# STEP 1: Import libraries
		from sentence_transformers import SentenceTransformer
		import faiss
		import numpy as np
		from openai import OpenAI


		# ==========================================
		# STEP 2: Load Embedding Model
		# ==========================================

		print("Loading embedding model...")

		embedding_model = SentenceTransformer(
		    "sentence-transformers/all-MiniLM-L6-v2"
		)


		# ==========================================
		# STEP 3: Create Knowledge Base Documents
		# ==========================================

		documents = [
		    "Python is a popular programming language used for AI and web development.",
		    
		    "Machine Learning is a field of Artificial Intelligence where computers learn from data.",
		    
		    "FAISS is a vector database library created for fast similarity search.",
		    
		    "all-MiniLM-L6-v2 converts text into 384 dimensional embeddings.",
		    
		    "RAG means Retrieval Augmented Generation where documents are retrieved before answering."
		]


		# ==========================================
		# STEP 4: Convert Documents to Embeddings
		# ==========================================

		print("Creating document embeddings...")

		document_vectors = embedding_model.encode(
		    documents
		)


		print("Embedding shape:")
		print(document_vectors.shape)


		# ==========================================
		# STEP 5: Create Vector Database
		# (FAISS)
		# ==========================================


		dimension = 384


		vector_db = faiss.IndexFlatL2(
		    dimension
		)


		vector_db.add(
		    np.array(document_vectors)
		    .astype("float32")
		)


		print(
		    "Documents stored:",
		    vector_db.ntotal
		)



		# ==========================================
		# STEP 6: User Question
		# ==========================================

		question = input(
		    "\nAsk question: "
		)



		# ==========================================
		# STEP 7: Convert Question into Vector
		# ==========================================


		query_vector = embedding_model.encode(
		    [question]
		)



		# ==========================================
		# STEP 8: Search Similar Documents
		# ==========================================


		top_k = 3


		distance, indexes = vector_db.search(
		    np.array(query_vector)
		    .astype("float32"),
		    top_k
		)



		# Get matching documents

		results = []


		for i in indexes[0]:
		    results.append(
			documents[i]
		    )


		print("\nRetrieved Documents:")
		for doc in results:
		    print("-", doc)



		# ==========================================
		# STEP 9: Create Prompt for LLM
		# ==========================================


		context = "\n".join(results)


		prompt = f"""

		Use the context below to answer the question.

		Context:
		{context}


		Question:
		{question}

		Answer:
		"""



		# ==========================================
		# STEP 10: Send to GPT
		# ==========================================


		client = OpenAI(
		    api_key="YOUR_OPENAI_API_KEY"
		)



		response = client.chat.completions.create(

		    model="gpt-4.1",

		    messages=[
			{
			    "role":"user",
			    "content":prompt
			}
		    ]

		)



		# ==========================================
		# STEP 11: Final Answer
		# ==========================================


		answer = response.choices[0].message.content


		print("\nFinal Answer:")
		print(answer)		
		
		
bert-base-uncased :


	- bert-base-uncased is a Transformer-based language understanding model, not a text generation model like GPT. 
	  Its job is to understand the meaning and context of text by converting words into numerical representations (embeddings/hidden states) that capture language patterns.
	  
	- It is mainly used for tasks like text classification, question answering, named entity recognition, and feature extraction.	
	

	Breaking down the name :
	
		BERT : 
		
			- Stands for Bidirectional Encoder Representations from Transformers.
			- Created by Google.
			- Uses the Transformer encoder architecture.
			- "Bidirectional" means it reads text by looking at both left and right context.	
			
			

OpenAI embedding models :

			- OpenAI embedding models convert text (or other supported inputs) into vectors (lists of numbers) that capture meaning. 
			  These vectors are used for things like semantic search, RAG (retrieval-augmented generation), clustering, recommendations, and similarity matching.	
			  
			  
			  
			  Common uses:

					Semantic search :
					
						- Find documents by meaning, not exact keywords.
						
					RAG systems :
					
						- Store document embeddings in a vector database like Pinecone, FAISS, or Chroma and retrieve relevant chunks before generating answers.
					
					Document similarity :
					
						- Compare articles, support tickets, or messages.
						
					Recommendation systems :
					
						- Match users/items based on meaning.
						
					Clustering :
					
						- Group similar text automatically.
